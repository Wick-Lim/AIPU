# rung-③ 어플라이언스 개념 스펙 — 512GB 전체 상주 박스 (v3, 2026-07-10)

**한 줄 요약.** GLM-5.2(UD-Q4_K_XL, ~467GB)를 **통째로 LPDDR5X 512GB에 상주**시키는
단일 사용자 오프라인 박스: **~76–95 tok/s [EST] 결정론 대역** (적응형 스펙체인 +
GLM-4.5-Air 실측 U 반영), ~40–60W, 보드 120×80mm,
BOM ~$1.8–2.4k. [`HARDWARE_LADDER.md`](HARDWARE_LADDER.md)의 rung-③ 메모리 피벗
(v2)을 실물 스펙으로 전개한 문서. 모든 성능/비용 수치는 **[EST]** — roofline +
h/U 측정(OLMoE 프록시 + GLM-4.5-Air 실측, [`H_MEASUREMENT.md`](H_MEASUREMENT.md))
기반이며, 실리콘/보드
실측이 아니다. 개념 배치도는 세션 아티팩트(요청 시 재생성 가능)로 유지.

## 1. 시스템 구성 (FIXED)

```
[AIPU SoC (우리 RTL)] ←1024-bit · ~1.1TB/s→ [LPDDR5X 512GB: 체크포인트 전체 상주]
        ↑ PCIe x4 — 부팅 시에만
[M.2 2230 NVMe 1TB: 콜드 보관] — 전원 인가 후 467GB 로드 ~70초, 이후 관여 없음
        ↑ USB4 (USB-C) — 토큰/프롬프트만. 전력은 별도(아래 §4)
[호스트 PC/폰]
```

- 스트리밍 티어(ONFI 64ch NAND)는 **삭제** — 상주 피벗의 정의. 하이브리드
  1024-bit(~84 tok/s, 단 GLM h≥0.75 전제)는 **같은 다이의 업사이드 SKU**로 보류
  (ONFI 패드는 다이에 유지, 상주 SKU 패키지에서 미배선).
- 컴퓨트 다이 = 지금 검증된 RTL(46.5MHz FPGA에서 1155-test 비트 정확)의 스케일업.
  expert-cache/prefetcher는 상주 구성에서 지연 은닉용으로 강등, weight_loader는
  DRAM 타일 페치로 단순화.

## 2. 성능 산출 (min(NAND, DRAM) 교정 반영)

토큰당 가중치 터치 (K=4 스펙체인, A≈3, U(4)≈2.4 — 프록시 측정):
`14·U/A (routed) + 11/A (핫셋) + ~0.5 (KV) ≈ 15.4GB — h와 무관.`

**tok/s = 1.1TB/s ÷ 15.4GB ≈ 71 [EST], 결정론** (h 불확실성이 설계에서 제거됨).
**갱신(GLM-4.5-Air 실측 U):** 적응형 스펙체인 포함 실효 대역 **~76–95 tok/s**
— 남은 변수는 수락률 r 하나 ([`H_MEASUREMENT.md`](H_MEASUREMENT.md) 2차 측정).

## 3. 실리콘: 클록은 목표가 아니라 파생값

불변량: `lane 수 × 클록 ≥ 스트림 소화율`. 한계점 총 소비 ~650GB/s ≈ **1.15T
weights/s** (Q4_K ~4.5bit/wt)에서, 마진 1.3× 포함:

| lane | 필요 클록 | 판정 |
|---|---|---|
| 1,024 | ~1.4GHz | 타이밍 빡빡 |
| 2,048 | ~730MHz | 가능 |
| **~3,072** | **~490MHz** | **스윗스팟** |

- **노드 선택의 바닥은 코어 클록이 아니라 LPDDR5X PHY IP** (벤더 제공 7–16nm)
  → **12–16nm + 코어 ~500MHz + lane ~3K**. 1GHz 불필요.
- lane 면적: dequant+4MAC ≈ 2–3만 게이트 × 3K ≈ 다이 수 mm² — SRAM/PHY가 다이 지배.
- FPGA 캠페인의 비트 정확 스테이지 분해(rope 10단/act 20단/matmul 5단)가 그대로 이월.

## 4. 전력 (v1→v3 개정 이력 — 정직하게)

| 판 | 총 전력 | 오류/변경 |
|---|---|---|
| v1 | ~35–55W | **NAND 읽기 에너지 과소평가** |
| v2 | ~70–120W | 물리 교정: NAND 읽기 ~25–55pJ/bit → 200GB/s 지속 = **40–90W가 열 예산 지배** (SSD 실측 ~100–135pJ/bit에서 컨트롤러/PCIe 제거분 차감) |
| **v3** | **~40–60W** | 상주 피벗으로 NAND 스트림 항목 자체가 소멸. LPDDR(~1.1TB/s, 4–6pJ/bit)가 새 지배 레일 ~25–40W + SoC 15–25W |

- **외부 전원 필수**: DC 19V 또는 USB-PD (v3면 100–140W로 충분). **PC의 USB-C
  포트는 7.5–15W 소싱이라 불가** — 데이터(USB4)와 전력은 분리 계통.
  옵션: 15W 버스파워 "여행 모드"(스로틀, 한 자릿수 tok/s).
- VRM ~12상: VCORE / VDDQ(LPDDR — v3의 최대 레일) / 보조 분담. 다상 병렬로
  저전압·대전류(레일당 수십 A)의 발열 분산 + 리플 상쇄 + 부하 과도응답.

## 5. 패키징/보드 — 소형화의 열쇠

- **1024-bit 버스를 SoC 패키지 서브스트레이트 안에서 완결** (LPDDR5X 32GB ×16을
  다이 주위에 온-서브스트레이트 실장 — Apple M-Ultra가 양산 실증한 관행).
  광폭 고속 배선이 PCB로 나오지 않으므로 **보드는 8층 · ~120×80mm**로 충분
  (v2의 NAND 뱅크 32팩 + 150×105mm 대비 면적 −40%).
- SoC 패키지 ~80×62mm; BGA ~1,400볼(전원 + PCIe x4 + USB만 — 1024-bit 미포함).
- 뒷면: M.2 2230 + VRM 일부. 방열: 베이퍼챔버 + 저소음 팬 (40–60W급).
- 이 패키징(온-서브스트레이트 16팩)이 **프로젝트 최대 난이도의 하드웨어 항목** —
  관행은 실증돼 있으나 우리가 아직 안 한 일 [PENDING].

## 5b. 공급망 노트 — 32GB LPDDR5X 패키지 의존성

512GB = **32GB 패키지 ×16**은 삼성 10.7Gbps LPDDR5X(2024-04 발표, 하반기 양산;
8단 32GB 초박형 2024-08 양산)로 실존하며 Apple M4 Max 128GB(4팩)가 실기 증거.
단 모바일 DRAM은 **B2B/NDA 유통**(공개 카탈로그 부재)이라 조달은 벤더 직접
협상 [PENDING]. 리스크 완충: 32GB 조달 실패 시 24GB×16=384GB로는 상주가
깨지므로(467GB 미달) → 하이브리드 SKU(동일 다이, ONFI 패드)로 회귀; 테이프아웃
시점('27±)엔 32GB 커머디티화 + LPDDR6 세대 진입으로 리스크 자연 감소 전망.

## 7. 경쟁 포지셔닝 — "체급"을 정확히

**사용자당 출력 속도, 같은 브래킷 비교** (API 수치는 시점/부하에 따라 변동):

| | 사용자당 tok/s | 비고 |
|---|---|---|
| 사람 읽는 속도 | ~7–12 | 기준선 |
| Claude Opus급 API | ~25–60 | 네트워크+큐 지연 별도 |
| Gemini Pro급 API | ~40–90 | 〃 |
| Mac Studio M3 Ultra 512GB ($10k+) | ~15–25 (이 급 모델, llama.cpp) | 현존 동브래킷 최강 |
| **AIPU v3 [EST]** | **~76–95 — 결정론 대역** | 첫 토큰 지연 無, 피크타임 저하 無 |

- **Groq/Cerebras(500+ tok/s)는 다른 종목**: 가중치를 SRAM에 상주(칩당 ~230MB)
  → 70B급에 칩 수백 개, 랙당 $수백만, 수천 사용자 분할상환. 467GB 모델은
  싱글박스 자체가 불가. 우리 브래킷(1인·1박스·풀 프런티어 모델·<$5k)에서
  500 tok/s는 물리적으로 성립하지 않음(467GB SRAM 불가; HBM 3TB/s는 $수만).
- 피치: **"프런티어급 오픈 모델을 Opus/Gemini-Pro 속도로, 오프라인에서, 구독
  없이"** — 동브래킷 최강(Mac Studio) 대비 ~3–4× 속도를 절반 가격에.

## 8. 기각된 대안들 (근거 박제)

- **CXL 512GB**: 링크당 실효 ~50GB/s(x16 Gen5) = 문헌의 PCIe 오프로딩 벽 →
  ~4–5 tok/s, $/GB ~30×, CXL 루트는 서버급 실리콘. 용량은 풀고 대역폭은 못 풂.
- **512GB DDR5 DIMM 상주**: ~350–540GB/s → ~25–35 tok/s. 용량을 느린 매체에
  강제. 서버 폼팩터.
- **256GB 하이브리드 (v1 고정안)**: min(NAND,DRAM) 교정 후 512-bit ~42 tok/s;
  1024-bit ~84는 GLM h≥0.75 베팅(빗나가면 ~45) — 업사이드 SKU로만 보류.

## 9. 성능/비용 레버 감사 (2026-07-10 결정)

v3 기준(~71 tok/s)에서 "더 빠르고 더 싸게"의 레버 전수 감사 결과:

- **✅ 채택 — 적응형 스펙체인 (RTL 반영 완료 — LANDED, commit 6c5332f).** 측정
  U(K) (GLM-4.5-Air 실측, 워크로드 평균)로 K-스윕:

  | K | U (GLM-Air 실측) | tok/s @r=0.8 | @r=0.85 | @r=0.9 |
  |---|---|---|---|---|
  | 1 | 1.00 | 78 | 80 | 82 |
  | 3 | 2.16 | 78 | 84 | 91 |
  | 4 | 2.65 | 76 | 84 | 93 |
  | 5 | 3.11 | 74 | 83 | **94** |
  | 8 | 4.27 | 67 | 79 | 95 |

  결론: **최적 깊이는 수락률 r의 함수** — r=0.8이면 K=2–3, r=0.9면 K=4–5에서
  고원(~94, K>5 무이득 → 적응 범위 K∈[1..5]로 하드웨어 단순화). 고정 K가
  아니라 **런타임 수락률 관찰 → 깊이 조절(적응형)**이 설계 정답. spec==greedy 속성이 모든 K에서 성립하므로 적응은
  **출력 불변**(스루풋만 변화) — 비트 정확 무트와 무충돌.
  **(updated 2026-07: LANDED — commit 6c5332f.** spec_decode_seq에 ADAPT
  파라미터(기본 0 — 기존 소비자는 yosys 순차 등가 PROVEN 무변경) + k_cur 포트 +
  pass_* 탭; 신규 `src/spec_depth_adapt.v` 적응 정책(포화 스트릭); 임의 깊이
  스케줄에서 spec==greedy — 구성상 출력 불변. 게이트: spec_depth_adapt 31,522 ·
  spec_decode_seq(K>1) 3,702 (K=1/2/3/4/6/8) · K=1 정확 621 · spec_chain_top 4/4
  (신규 DRAFT_K=4 엔진 포함) · spec_batched_top 8/8 · spec_decode_top 18/18 ·
  `make spec-adapt` 게이트 신설.)
  남은 결정 변수는 **GLM의 r 실측** 하나 (job B: vLLM MTP 스윕, 진행 중 —
  GLM-5.2 저지연 모드 공표치 4+ 수락은 r~0.9 시사).
- **📋 계획 전용 — LPDDR6.** 같은 16팩·같은 폭에서 BW ~1.5× → ~105 tok/s.
  테이프아웃 시점('27±)에 성숙 예상 — PHY IP 교체만으로 얻는 세대 레버라
  **능동 작업 없음, 로드맵 항목으로만 유지.** (적응형 스펙체인과 합산 시
  ~130+ [EST].)
- ⏸ 보류: 핫셋 HBM 분리(+$300–500, 2.5D 패키징 점프 — LPDDR6가 더 싸게 동일
  효과), LPDDR-PIM(생태계 미성숙, 관찰만).
- ✗ 기각: expert 저비트화(비트 정확 무트 파괴), 로직 칩렛 분리(NRE↓가 패키징↑로
  상쇄), 메모리 하위 빈(성능↓).
