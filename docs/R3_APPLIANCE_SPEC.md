# rung-③ 어플라이언스 개념 스펙 — 512GB 전체 상주 박스 (v3, 2026-07-10)

**한 줄 요약.** GLM-5.2(UD-Q4_K_XL, ~467GB)를 **통째로 LPDDR5X 512GB에 상주**시키는
단일 사용자 오프라인 박스: 설계점 **≈ 80 tok/s [실측-입력 EST]** (적응형 스펙체인 +
GLM-4.5-Air 실측 U(K)·실측 수락률 r(잡 B) 반영; GLM-5.2의 더 깊은 MTP가 공표
수준이면 ~95 — 이전 표기 ~76–95 대역의 실측 후 갱신), ~40–60W, 보드 120×80mm,
BOM ~$1.8–2.4k. [`HARDWARE_LADDER.md`](HARDWARE_LADDER.md)의 rung-③ 메모리 피벗
(v2)을 실물 스펙으로 전개한 문서. 모든 성능/비용 수치는 **[EST]** — roofline +
h/U 측정(OLMoE 프록시 + GLM-4.5-Air 실측, [`H_MEASUREMENT.md`](H_MEASUREMENT.md))
기반이며, 실리콘/보드
실측이 아니다. 개념 배치도는 세션 아티팩트(요청 시 재생성 가능)로 유지.

## 1. 시스템 구성 (FIXED)

```
[AIPU SoC (우리 RTL)] ←1024-bit · ~1.1TB/s→ [LPDDR5X 512GB: 체크포인트 전체 상주]
        ↑ PCIe x4 — 부팅 시에만
[M.2 2230 NVMe 1TB: 콜드 보관] — 전원 인가 후 467GB 로드 ~70초, 이후 관여 없음
        ↑ USB4 (USB-C) — 토큰/프롬프트만. 전력은 별도(아래 §4)
[호스트 PC/폰]
```

- 스트리밍 티어(ONFI 64ch NAND)는 **삭제** — 상주 피벗의 정의. 하이브리드
  1024-bit(~84 tok/s, 단 GLM h≥0.75 전제)는 **같은 다이의 업사이드 SKU**로 보류
  (ONFI 패드는 다이에 유지, 상주 SKU 패키지에서 미배선).
- 컴퓨트 다이 = 지금 검증된 RTL(46.5MHz FPGA에서 1155-test 비트 정확)의 스케일업.
  expert-cache/prefetcher는 상주 구성에서 지연 은닉용으로 강등, weight_loader는
  DRAM 타일 페치로 단순화.

## 2. 성능 산출 (min(NAND, DRAM) 교정 반영) — **토큰당 바이트 상수의 정본(canonical)**

**토큰당 바이트 상수 — 정본 유도 (다른 문서는 이 값을 인용한다; 상이한 수치는
시나리오 라벨 필수):**

- **라우티드 expert 스트림 ≈ 14 GB/token**: 75 MoE층 × top-8 × 37.75M
  params/expert = 22.65B params/token, × **~0.62 B/param** (UD-Q4_K_XL
  467 GB ÷ 753B) ≈ **14 GB**. 스펙체인에서는 검증 패스당 expert *합집합*이
  ×U(K)로 성장하고 수락 A로 상각 → `14·U(K)/A`.
- **핫셋 터치 ≈ 11 GB/token**: 토큰당 활성 ~40B params ≈ ~25 GB에서 라우티드
  14 GB를 뺀 나머지(어텐션 투영 전층 + dense FFN + 공유 expert + router +
  LM-head) — 검증 패스당 1회 읽혀 A로 상각 → `11/A`.
  *(구분: DDR에 **상주해야 하는 핫 파티션 용량**은 비-라우티드 ~28B params ≈
  **~17 GB** (embed/LM-head/norm 포함) — roofline이 나누는 것은 상주 용량이
  아니라 토큰당 터치 11 GB다. 구문서의 "~9 GB 핫셋"은 이 교정 전의 저계상
  추정치로, 본 문서가 대체한다.)*
- **KV ≈ 0.5 GB/token.**

토큰당 가중치 터치 (K=4 스펙체인, A≈3, U(4)≈2.4 — 프록시 측정):
`14·U/A (routed) + 11/A (핫셋) + ~0.5 (KV) ≈ 15.4GB — h와 무관.`

**tok/s = 1.1TB/s ÷ 15.4GB ≈ 71 [EST], 결정론** (h 불확실성이 설계에서 제거됨).
**갱신 2(r까지 실측, 잡 B):** MTP 수락률 실측 r₁=0.87·위치별 급감 →
메모리-바운드 최적 깊이 K=1–2, **상주 박스 ≈ 80 tok/s [실측-입력 EST]**
(GLM-5.2의 더 깊은 MTP가 공표 수준이면 ~95; [`H_MEASUREMENT.md`](H_MEASUREMENT.md)
3차 측정). 적응형 컨트롤러는 이 최적점에 자율 안착 — 설계 검증됨.

## 3. 실리콘: 클록은 목표가 아니라 파생값

불변량: `lane 수 × 클록 ≥ 스트림 소화율`. 한계점 총 소비 ~650GB/s ≈ **1.15T
weights/s** (Q4_K ~4.5bit/wt)에서, 마진 1.3× 포함:

| lane | 필요 클록 | 판정 |
|---|---|---|
| 1,024 | ~1.4GHz | 타이밍 빡빡 |
| 2,048 | ~730MHz | 가능 |
| **~3,072** | **~490MHz** | **스윗스팟** |

- **노드 선택의 바닥은 코어 클록이 아니라 LPDDR5X PHY IP** (벤더 제공 7–16nm)
  → **12–16nm + 코어 ~500MHz + lane ~3K**. 1GHz 불필요.
- lane 면적: dequant+4MAC ≈ 2–3만 게이트 × 3K ≈ 다이 수 mm² — SRAM/PHY가 다이 지배.
- FPGA 캠페인의 비트 정확 스테이지 분해(rope 10단/act 20단/matmul 5단)가 그대로 이월.

## 4. 전력 (v1→v3 개정 이력 — 정직하게)

| 판 | 총 전력 | 오류/변경 |
|---|---|---|
| v1 | ~35–55W | **NAND 읽기 에너지 과소평가** |
| v2 | ~70–120W | 물리 교정: NAND 읽기 ~25–55pJ/bit → 200GB/s 지속 = **40–90W가 열 예산 지배** (SSD 실측 ~100–135pJ/bit에서 컨트롤러/PCIe 제거분 차감) |
| **v3** | **~40–60W** | 상주 피벗으로 NAND 스트림 항목 자체가 소멸. LPDDR(~1.1TB/s, 4–6pJ/bit)가 새 지배 레일 ~25–40W + SoC 15–25W |

- **외부 전원 필수**: DC 19V 또는 USB-PD (v3면 100–140W로 충분). **PC의 USB-C
  포트는 7.5–15W 소싱이라 불가** — 데이터(USB4)와 전력은 분리 계통.
  옵션: 15W 버스파워 "여행 모드"(스로틀, 한 자릿수 tok/s).
- VRM ~12상: VCORE / VDDQ(LPDDR — v3의 최대 레일) / 보조 분담. 다상 병렬로
  저전압·대전류(레일당 수십 A)의 발열 분산 + 리플 상쇄 + 부하 과도응답.

## 5. 패키징/보드 — 소형화의 열쇠

- **1024-bit 버스를 SoC 패키지 서브스트레이트 안에서 완결** (LPDDR5X 32GB ×16을
  다이 주위에 온-서브스트레이트 실장 — Apple M-Ultra가 양산 실증한 관행).
  광폭 고속 배선이 PCB로 나오지 않으므로 **보드는 8층 · ~120×80mm**로 충분
  (v2의 NAND 뱅크 32팩 + 150×105mm 대비 면적 −40%).
- SoC 패키지 ~80×62mm; BGA ~1,400볼(전원 + PCIe x4 + USB만 — 1024-bit 미포함).
- 뒷면: M.2 2230 + VRM 일부. 방열: 베이퍼챔버 + 저소음 팬 (40–60W급).
- 이 패키징(온-서브스트레이트 16팩)이 **프로젝트 최대 난이도의 하드웨어 항목** —
  관행은 실증돼 있으나 우리가 아직 안 한 일 [PENDING].

## 5a. RTL ↔ SKU 매핑 (2026-07-10, RESIDENT 파라미터 랜딩)

메모리-시스템 RTL은 **한 벌로 세 구성**을 섬긴다 — 이름 일부는 스트리밍
시절의 유물이지만 기능은 티어-불문이며, 상주 구성은 `RESIDENT=1` 파라미터로
RTL 수준에서 검증된다 (`make resident`, 19 tests: 양 모드 DUT 대조 —
RESIDENT=1에서 expert 리필이 ddr5_xbar의 태그된 읽기(TAG_EFILL)로 서빙되고
flash 경로는 시뮬 불변식 `$fatal`로 발화 불가 증명; RESIDENT=0은 기존과 배선
동일 — 상수 폴딩 삼항 + 타이오프 generate, **yosys 순차 등가검증으로 형식
증명** `make resident-equiv`).

| RTL 블록 | rung-① 데모 (스트리밍) | **상주 주 SKU (RESIDENT=1)** | 하이브리드 SKU |
|---|---|---|---|
| `ddr5_xbar` | DDR4/5 fast-tier 팹릭 | **LPDDR5X 팹릭** (이름은 유물, 기능은 티어-불문 크로스바) | LPDDR5X 팹릭 |
| `weight_loader_q4k` | 스트림/캐시 타일 페치 | DRAM 타일 페치 | 양쪽 |
| `expert_cache_pf/ctrl` | h-캐시 본연 | 리필=DDR 읽기 (`glm_q4k_system.v` §10 FSM), hit/miss 의미 불변 | h-캐시 본연 |
| `flash_xbar` + FLASH 채널 | NVMe expert 스트림 | **KV NVMe-스필 전용** (expert 클래스 발화 불가) | expert 스트림 복원 |
| `boot_loader` | 〃 | **NVMe→DRAM 70초 부팅** (형식 증명 완료) | 〃 |
| `kv_cache_pager` | KV 링 + flash 오버플로 | KV 링 + **NVMe 스필** ("무조건 캐싱" 결정의 하드웨어) | 〃 |
| `em_/gn_` 포트 (GDDR6 스텁) | 유휴 | 유휴 — **구세대 컨셉 유물, 테이프아웃 시 제거 후보** | 유휴 |

## 5b. 공급망 노트 — 32GB LPDDR5X 패키지 의존성 (2026-07-11 유통 실사 반영)

512GB = **32GB 패키지 ×16**은 삼성 10.7Gbps LPDDR5X(2024-04 발표, 하반기 양산;
8단 32GB 초박형 2024-08 양산)로 실존하며 Apple M4 Max 128GB(4팩)가 실기 증거.
단 모바일 DRAM은 **B2B/NDA 유통**(공개 카탈로그 부재)이라 조달은 벤더 직접
협상 [PENDING]. **유통 실사(2026-07-11, 웹 확인):** 소매/애프터마켓(폰 수리
부품 시장)에서 실구매 가능한 패키지 상한은 **24GB**(SK hynix H58GJ6BV9HX151R,
9.6Gbps, 496-FBGA — Alibaba/AliExpress 재고 확인; 12GB/16GB도 상시), **32GB는
확인 실패** — OEM 완제품 내장 또는 Micron SOCAMM2(패키지당 64GB, 하이퍼스케일러
전용 모듈)로만 존재. 리스크 완충: 32GB 조달 실패 시 24GB×16=384GB로는 상주가
깨지므로(467GB 미달) → **§5c 프로토 설계점(24GB×20)** 또는 하이브리드 SKU(동일
다이, ONFI 패드)로 회귀; 테이프아웃 시점('27±)엔 32GB 커머디티화 + LPDDR6 세대
진입으로 리스크 자연 감소 전망.

## 5c. v3-proto — 소매부품 프로토 설계점 (2026-07-11 결정: 프로토 우선)

**결정.** rung-③ 실행 순서를 "양산 SKU 먼저"에서 **"소매 조달 가능한 프로토
먼저"로 재배열.** §5b 실사로 32GB NDA 조달과 온-서브스트레이트 16팩(§5 —
프로젝트 최대 난이도 항목)이 프로토 크리티컬 패스에서 빠지고, 실리콘만 나오면
나머지는 즉시 조립 가능해진다.

| | **v3-proto (프로토, 소매 조달)** | v3-volume (양산 SKU) |
|---|---|---|
| DRAM | **24GB ×20 (SK hynix 9.6Gbps, 소매)** | 32GB ×16 (벤더 협상) |
| 용량 | 480GB (467GB 모델 + ~13GB KV 링; KV는 NVMe-스필이 흡수) | 512GB |
| 버스 | **1280-bit** (20팩 × 64-bit) | 1024-bit (같은 다이, 4채널그룹 미배선) |
| 대역폭 | ~1.54TB/s → **~110 tok/s [EST]** | ~1.1–1.37TB/s → ~80–100 [EST] |
| 실장 | **PCB-HDI 직실장** (폰 관행: 8.5–9.6Gbps를 PCB 수 cm 라우팅), **12층**(신호6·GND4·전원2 — SoC escape ~2,800신호 ÷ 층당 ~460 = 신호층 6 필요; 10층은 전원 평면 1장으로 PI 미달), 보드 ~130×110mm | 온-서브스트레이트 16팩, 보드 120×80mm |
| 다이 | 1280-bit PHY beachfront ~75–85mm → **패드-리미티드 ~480mm²** [EST] — 12–16nm에서 원가 ~$40–60, BOM 허용 범위 | 같은 다이 (수퍼셋) |

- **다이는 하나다**: 1280-bit로 파두면 1024-bit 구성을 포용(남는 채널 미배선)
  — 프로토와 양산 SKU가 같은 실리콘, 마스크 재사용.
- **프로토 SoC 패키지 볼아웃 [EST]**: 버스가 보드로 나오므로 신호 ~2,800
  (DQ 1,280 + WCK/RDQS/DMI ~800 + CA/CK/CS ~800) + 전원/GND → **0.8mm 피치
  ~4,300볼, ~50×50mm** — 서버 CPU급(AMD SP5 6,096볼) 관행 안. §5의 ~1,400볼은
  버스가 패키지 안에서 끝나는 양산형 수치로, 프로토엔 적용되지 않음.
- **정직한 한계**: 이 피벗이 없애는 것은 *패키징·조달* 리스크다. **SoC
  테이프아웃 자체(LPDDR5X PHY IP 라이선스 + 12–16nm NRE)는 프로토라고
  사라지지 않는다** — rung-③의 진짜 관문은 여전히 실리콘 [PENDING].
- 12GB×40(2-rank 1280-bit, 480GB) 극단은 DRAM 최저가지만 40팩 실장이 보드를
  서버-미니급으로 키우고 그레이마켓 QC 리스크가 팩 수에 비례 — 기각, 근거 박제.
- 24GB 소매 파트가 9.6Gbps라 기존 v3 계산(8.5Gbps, 1.1TB/s)보다 **대역폭이
  오히려 상향**된다(1024-bit 기준으로도 1.23TB/s ≈ ~89 tok/s [EST]).
- **전력 [EST]**: 메모리-바운드 설계라 전력은 대역폭에 비례 — 1.54TB/s는 v3의
  1.4×라 DRAM 레일 ~35–55W + SoC 15–25W = **~50–80W** (v3-volume ~40–60W,
  §4). 냉각 방식(베이퍼챔버+팬)은 동일 급.

| | 사용자당 tok/s | 비고 |
|---|---|---|
| 사람 읽는 속도 | ~7–12 | 기준선 |
| Claude Opus급 API | ~25–60 | 네트워크+큐 지연 별도 |
| Gemini Pro급 API | ~40–90 | 〃 |

- **Groq/Cerebras(500+ tok/s)는 다른 종목**: 가중치를 SRAM에 상주(칩당 ~230MB)
  → 70B급에 칩 수백 개, 랙당 $수백만, 수천 사용자 분할상환. 467GB 모델은
  싱글박스 자체가 불가. 우리 브래킷(1인·1박스·풀 프런티어 모델·<$5k)에서
  500 tok/s는 물리적으로 성립하지 않음(467GB SRAM 불가; HBM 3TB/s는 $수만).

## 8. 기각된 대안들 (근거 박제)

- **CXL 512GB**: 링크당 실효 ~50GB/s(x16 Gen5) = 문헌의 PCIe 오프로딩 벽 →
  ~4–5 tok/s, $/GB ~30×, CXL 루트는 서버급 실리콘. 용량은 풀고 대역폭은 못 풂.
- **512GB DDR5 DIMM 상주**: ~350–540GB/s → ~25–35 tok/s. 용량을 느린 매체에
  강제. 서버 폼팩터.
- **256GB 하이브리드 (v1 고정안)**: min(NAND,DRAM) 교정 후 512-bit ~42 tok/s;
  1024-bit ~84는 GLM h≥0.75 베팅(빗나가면 ~45) — 업사이드 SKU로만 보류.
