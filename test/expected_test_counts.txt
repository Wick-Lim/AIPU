# expected_test_counts.txt -- EXACT per-gate test-count manifest (enforced by
#   tools/check_test_counts.sh, wired as `make release-gate-strict`).
#
# Each gate recipe greps `ALL [0-9]+ TESTS PASSED` -- a wildcard that still passes if a
# testbench silently ran FEWER tests than intended (a reduced loop bound, a dropped
# `ifdef`, a missing build flag such as -DSPEC_FULL).  This manifest pins the EXACT count
# each gate must print; the checker fails on any mismatch or a gate that never printed.
#
# FORMAT:  NAME  c1 [c2 ...]
#   NAME  = the exact string inside the gate's `[...]` banner prefix (no spaces).  A unit
#           exercised under two configs prints two DISTINCT banner names when the recipe
#           tags them (e.g. moe_router_q4k vs moe_router_q4k(256/top-8)) -- those are
#           separate lines here.
#   c1..  = the expected test count(s).  A few names print MORE THAN ONCE with the SAME
#           banner (the same recipe run twice, e.g. inside two meta-targets) -- for those
#           the full MULTISET of counts is pinned, and the checker compares them as
#           histograms so a drop/add of any single run, or a change to any one count, fails.
#
# MAINTENANCE: when you add or remove test cases (so a gate's printed count changes), or
#   add/remove a gate invocation, update the matching line here.  That is the intended
#   friction: a test count is a contract, and a silent change to it is a regression.
#
# Captured from a GREEN `make release-gate` (2026-07, exit 0, "release-gate: ALL gates
#   passed").  Regenerated with the checker's own banner parser so the manifest and checker
#   agree by construction (distinct parenthetical config-variant names kept separate).
#   Non-"ALL N TESTS PASSED" gates (synth-glm cell counts, formal unsat, cdc, the
#   structural-equiv gates and the must-FAIL injection steps) are intentionally absent --
#   they do not emit this banner and are checked by their own recipes.  Every gate that
#   DOES emit the banner must be pinned here: the checker now FAILS on any countable
#   banner without a manifest line (2026-07-22 -- dsa-sparse-correct, boot_loader_manifest
#   and host-test had banner formats the parser could not see, and ran count-unchecked;
#   their banners are normalized and pinned below).
