The published GGUF k-quant of GLM-5.2 (UD-Q4_K_XL, ~467 GB, a 753B-param MoE, ~40B active/token) streamed through a Verilog datapath. The Q4_K GEMM core is bit-exact to an independent ggml reference; the model is wrapped by a multi-channel DDR5 + NVMe memory system with formal proofs on the control plane. Single-user, air-gapped: provisioned once with the weights, then run with no network path in or out.

Throughput

Every tok/s figure is modeled [EST], not measured on silicon. The amortization mechanism is measured in RTL (the hardware weight-load counter); the accept rate feeding it is a measured model property; the absolute tok/s is their product.

How the number is built

25.50 GB/token — no speculation, PE_M=1 (measured mechanism)
÷ A_eff = 1.87 — measured accept rate r₁=0.87
= 13.87 GB/token
→ at 1.54 TB/s → ~111 tok/s [EST]

The composed top glm_q4k_spec_system holds the memory system, the PE_M=K+1 batched verify, and the accept/reject loop at once. Its hardware counter shows ALL-ACCEPT committing exactly K+1 tokens per weight-load (K=3 → 4.00), and the committed stream equals a plain greedy decode.

Rung ladder — set by memory bandwidth

Throughput is bandwidth-bound, and bandwidth is set by the chip’s IO/PHY budget, which is set by the rung.

  • ① Prove-it FPGA ~5–8 tok/s
  • ② Funded custom board ~15–40 tok/s
  • ③ 512GB residency box ≈80 tok/s

Rung-③ design point ≈80 tok/s [measured-inputs EST] — U(K) and the MTP accept rate r are both GLM-family measured. ~95 if GLM-5.2’s deeper MTP hits its published accept depth.

How it runs

NVMe/PCIe-bandwidth-bound by design, to keep it cheap: stream MoE experts from the SSD through a DDR working cache into a mostly-idle Q4_K die.

1 TB NVMe ──► flash_xbar ──► DDR working set/cache ──► ddr5_xbar ──► Q4_K compute die ──► token
~467 GB weights   N-ch + deep queue   LRU+freq+prefetch   N-ch banked   MLA + DSA + MoE · ggml dequant → fp32 MAC → bf16

The whole memory system is byte-agnostic — it moves addresses, slots, and IDs, never weight bytes. Verification is byte-identical on regression; the controllers add bounded model checking, some lifted to unbounded k-induction. No formal proof touches the numeric datapath — that plane is held by the bit-exact simulation goldens.

Not yet done

Open gaps — most sit at the physical/process boundary that an RTL repo cannot close.

Whole-runtime GGUF cross-check

The 467 GB GLM file run end-to-end and compared against llama.cpp’s runtime argmax. Needs a GPU / large-memory host; no in-repo tool yet.

Vendor PHY IP

DDR5 / NVMe (PCIe) / USB-C PHYs are TB-stubbed. Licensed PHY IP is integrated + signed off at the silicon rung.

Board bring-up

The FPGA fit is measured in-repo; running on a physical dev board needs the board + pin XDC + a MIG bridge.

Fab tapeout & dual-port BIST collar

ASIC tapeout is the rung-③ volume endgame. On-die test: the die already carries an inline clock gate; per-macro BIST collars are memory-compiler-generated in the physical flow.

Every tok/s, cost, energy, and LOI figure is [EST] — roofline-modeled with measured model-side inputs, not measured on hardware.
