프런티어 AI를 책상 위에서. 랜선을 뽑아도 돌아갑니다.

01

프런티어 AI는 연결을 요구합니다. 그런데 돈 되는 큰 시장은 연결할 수가 없습니다.

최고의 모델은 API 뒤에 있습니다. 그 API로 데이터를 보낼 수 없거나 — 변호사·의뢰인 비밀유지, 진료정보(PHI), 미공개 중요정보(MNPI), 기밀, 핵심 IP — 아예 연결이 끊긴 환경(SCIF, 격리된 OT 네트워크, 선박, 현장)이라면 아예 배제됩니다. "보안 클라우드"도 답이 아닙니다. VPC 내부, 무보존, 기밀 컴퓨팅(TEE) 모두 결국 연결이 필요합니다. 정작 중요한 하나의 시험을 통과하지 못합니다. 랜선을 뽑아도 돌아가는가?

누가 배제되는가

프런티어급 품질이 필요하지만 법·계약·기밀등급·데이터 주권·단절된 환경, 혹은 벤더 종속 금지 방침 때문에 반드시 오프라인/에어갭으로 돌려야 하는 모든 곳입니다. 법률, 퀀트/금융, 정부·국방, IP 민감 R&D, 헬스케어, 그리고 클라우드가 닿지 못하는 모든 현장·엣지·OT 사이트.

기존 선택지가 이들을 놓치는 이유

  • ✕클라우드 API — 연결이 필요하고, 데이터가 외부로 나갑니다.
  • ✕"보안" 클라우드(VPC / 무보존 / TEE) — 여전히 연결된 상태. 랜선 뽑기 시험을 통과하지 못합니다.
  • ✕GPU에 올린 70B 로컬 모델 — 오프라인이지만 품질 격차가 눈에 띄고, 관리할 장비가 됩니다.
  • ✕H100 8장으로 753B 자체 구축 — 약 $25만–40만 + 킬로와트급 전력 + MLOps. 완제품이 아니라 구축 프로젝트입니다.
02

제품: 데이터센터 가격이 아니라 좌석 가격에 담은 프런티어 모델

양자화된 모델 전체를 NVMe SSD에서 꺼내 빠른 DDR 캐시를 거쳐 Q4_K 연산 다이로 흘려보내는 단일 사용자 박스(FPGA 카드)입니다. 모델·캐시·연산 모두 박스 안에 있고, 호스트 링크는 토큰 ID만 주고받습니다. 그래서 랜선을 뽑아도 돌아갑니다. 일부러 메모리 대역폭에 묶어 값을 낮췄습니다. 467GB Q4_K 모델은 NVMe에 담고, 다이는 작게 유지하며, 속도는 값비싼 데이터센터 GPU가 아니라 메모리 대역폭에서 나옵니다. 성능은 하드웨어에 따라 단계별로 올라갑니다(아래 사다리 참고).

오프라인만으로는 기본기일 뿐입니다(노트북에 올린 70B도 오프라인입니다). 진짜 해자는 조합에 있습니다. 오프라인 + 온전한 프런티어 모델(753B) + 완제품/좌석 가격, 게다가 보안팀이 승인할 수 있는 결정적 실리콘. 467GB 가중치는 한 번 프로비저닝하면 되고(이 과정 자체도 오프라인으로 가능), 업데이트는 물리적 재프로비저닝입니다. 모델은 별도로 다시 만든 양자화가 아니라, 로컬 추론이 이미 돌리고 있는 형식인 공개 GGUF k-quant 그대로입니다.

성능은 메모리 대역폭 = 칩의 IO/PHY가 정합니다(연산은 값싸고, 다이는 메모리 시스템 뒤에서 놉니다). 그래서 정직하게 단계를 나눕니다. FPGA에서 싸게 증명 → 커스텀 보드 → 양산 ASIC. 모든 단계에서 같은 GGUF 모델을 쓰고, 실리콘이 공급하는 대역폭만 달라집니다. tok/s 수치는 대역폭 루프라인 추정치지만, 순수 추정이 아니라 실측한 U(K)와 MTP 수락률(GLM-4.5-Air에서 측정)을 입력으로 넣은 루프라인입니다. 아래 FPGA 데모가 이를 확정합니다.

03

검증된 RTL — 증명된 것, 엘라보레이션된 것, 그리고 남은 과제까지

검증 우선

대부분의 하드웨어 소개는 슬라이드에 그칩니다. 이것은 CI에서 게이팅되는 검증된 RTL입니다. Q4_K 연산은 GGML k-quant 레퍼런스와 비트 단위로 동일하고, 메모리 컨트롤러는 형식 검증을 통과했습니다. 전체 모델 포워드는 조립된 골든과 비트 단위로 동일하며(1,155 테스트), dequant 계층은 실제 공개 GGUF 두 파일에서 llama.cpp 자체 코드와 비트 단위로 동일합니다(가중치 3.77억 개, Q4_K/Q6_K/Q8_0). 아직 증명하지 못한 것도 증명한 것만큼 분명하게 밝힙니다. 보안팀의 승인을 받아야 하는 박스에는, 뭉뚱그린 주장이 아니라 정직한 검증 장부가 결론을 짓습니다.

"증명" 행은 모두 저장소에서 재현할 수 있습니다(make q4k, make formal, make model-q4k, make resident). dequant 계층에서 자기참조의 여지는 닫혔습니다. 우리 재구현은 실제 공개 GGUF 두 파일의 원시 블록에서 llama.cpp 자체 dequantize_row_q4_K/_q6_K와 비트 단위로 동일합니다(Q4_K/Q6_K/Q8_0, 가중치 3.77억 개). 따라서 이행성으로 RTL dequant ≡ 실제 파일의 ggml dequant. 열린 채로 남은 것은 감추지 않고 명시합니다. llama.cpp와의 전체 런타임 수치 동일성은 설계상 계약 밖입니다(fp32 누적 순서가 다르며, 우리 계약은 ggml과 동일한 dequant + 문서화된 자체 순서입니다).

04

실리콘으로 가는 길: FPGA 핏 관문

▍ 다음 관문

가장 큰 하드웨어 미지수 — "다이가 실제로 살 만한 FPGA에 들어가는가?" — 는 이제 추정이 아니라 실측입니다. Q4_K 제품 톱이 지목한 ~$1,000급 보드 부품에 배치·라우팅됩니다. 남은 것은 실물 보드 데모뿐입니다.

FPGA 핏이 박스의 BOM을 정하고, BOM이 사업 전체가 딛고 선 좌석당 가격을 정합니다. 연산 데이터패스는 753B 전체 규모로 엘라보레이션되고, 전체 칩은 구조 사인오프를 통과합니다(yosys check-assert, 미해결 0). 설계는 스케치가 아니라 처음부터 끝까지 일관됩니다.

목표 부품은 명시되어 있습니다. Kintex UltraScale+ XCKU3P(DSP48E2 1,368개, DDR4 + PCIe). 그리고 실제 만들 수 있는 증거의 가격도 정해져 있습니다. ~$1,000급 개발 보드이지, 연구 프로그램이 아닙니다. 벤더 툴 관문은 통과했습니다. Q4_K 제품 톱이 XCKU3P에서 배치·라우팅됩니다(LUT 87.5%, 비트 동일 리파이프라인 이후 라우팅 46.5 MHz). 핏은 고정된 커밋에서 실측했고, 이후로는 약속이 아니라 증명으로 이어갑니다. 이후의 RTL 변경은 핏 구성에서 넷리스트가 동일함을 형식 검증으로 증명하거나(yosys 순차 동치), 아니면 핏을 다시 돌립니다. 남은 것은 실제 가중치를 돌리는 보드 그 자체이며, 그 데모가 루프라인 추정치를 실측 tok/s로 바꿔 줍니다.

05

누가 먼저 사는가

ICP는 제약 조건이다 · 첫 진입점 = 법률

반드시 오프라인 — "보안 클라우드"로는 부족한 곳

ICP는 산업 분야가 아니라 제약 조건입니다. 프런티어급 품질이 필요하지만 반드시 에어갭으로 돌려야 한다는 것. 제품은 수평적이고, 첫 진입점은 좁습니다. 첫 진입점은 법률입니다. 클라우드에 닿을 수 없는 비밀유지 업무를 위해, 선임 변호사에게 개인 온프렘 박스를 한 좌석씩 지급하는 로펌. (가장 빨리 계약되는 퀀트도 병행 검증.)

  • 필요까다로운 계약·비밀유지·초안 작업에 프런티어급 품질.
  • 금지비밀유지 의무가 클라우드를 금지 — VPC 내부·TEE 포함.
  • 지불이미 Westlaw급 도구에 좌석당 월 $100–500+ 지출.
  • 속도몇 년이 아니라 몇 주 안에 움직이는 혁신/지식관리 구매자.
세그먼트제약 & 순서
퀀트 / 프롭공동 선두 — 지금 검증 · 가장 빠른 계약 · FPGA 친화 · 시그널이 새지 않음
오프라인 / 엣지에어갭 필수 · SCIF, 격리 OT/핵심 인프라, 현장·엣지 사이트
IP-R&D제약·반도체·코드 — 클라우드에 못 올리는 비밀, 실제 예산
정부 / 국방최고가치, 에어갭 필수 — 느리고, 검증된 모델 리타깃이 필요할 수 있음
헬스케어거대 시장, PHI 규제 — 위험 회피적이고 검증이 긴 편

이 제품의 메시지는 "더 싼 토큰"이 아니고(쓸 수만 있다면 클라우드가 더 쌉니다), "유출을 피하라"도 아닙니다(그건 아무것도 안 하는 선택과 경쟁합니다). 클라우드가 닿지 못하는 곳에서 프런티어 모델을 돌리는 유일한 턴키 방식 — 랜선을 뽑은 채, 에어갭으로, 온전히 내 것으로, 책상 가격에. 오프라인이 가능한 대안 — 같은 GGUF를 돌리는 대용량 RAM 워크스테이션 — 은 사인오프된 완제품이 아니라 직접 돌봐야 하는 구축물입니다.

정직하게 밝히는 한계

감추지 않고 기록
아직 실리콘 아님
처리량·BOM 수치는 루프라인 추정치입니다. 다만 연산 다이 자체는 이미 실제 라우팅된 FPGA 핏입니다(XCKU3P, 87.5% LUT, 비트 동일 리파이프라인 이후 라우팅 46.5 MHz). tok/s를 실측으로 바꾸는 것은 보드가 실제로 도는 데모입니다(§04, §06). 위에서 증명으로 표시한 것은 모두 저장소에서 재현할 수 있습니다.
런타임 동일성 범위 한정
조립된 753B 포워드는 골든과 비트 단위로 동일하고(1,155/1,155), dequant 계층은 실제 공개 GGUF 두 파일에서 llama.cpp 자체 코드와 비트 단위로 동일합니다(가중치 3.77억 개, Q4_K/Q6_K/Q8_0). 열린 채로 남은 것은 명시했습니다. 전체 런타임 llama.cpp 수치 동일성은 설계상 계약 밖입니다(fp32 누적 순서가 다름).
의도된 좁은 시장
프런티어 규모 + 반드시 오프라인이 동시에 걸리는 곳에서만 이깁니다. 그 교집합은 결함이 아니라 의도한 틈새입니다. Mac은 70B를 돌리고, 클라우드는 온라인으로 돌리며, 우리는 전체 모델을 랜선을 뽑은 채 돌립니다.
박스 하나, 좌석 하나
멀티유저 서버가 아니라 개인용 완제품(B=1)입니다. 배치/데이터센터 서빙은 같은 실리콘의 잠재 능력이지, 이 제품은 아닙니다.

설계상 모델에 종속되지 않습니다. 먼저 GLM-5.2(현시점 최상위 오픈웨이트 모델)에 맞춰 만들었지만, 박스는 FPGA입니다. 신뢰하는 어떤 프런티어 오픈웨이트 GGUF로든 리타깃할 수 있고, 이는 새 하드웨어가 아니라 RTL 작업입니다. 이전 FP8 데이터패스(FP8 체크포인트와 비트 단위 동일)는 저장소 이력의 fp8 브랜치에 보존되어 있습니다. 현재 제품 트랙은 Q4_K이며, GGUF k-quant가 로컬 추론의 네이티브 형식이고 디스크에서 약 38% 더 작기 때문입니다.

AIPU·로컬 프런티어 AI 박스
오프라인 프런티어 AI 박스 · GLM-5.2 Q4_K · [추정] 표시 수치는 라우팅된 실리콘이 아니라 루프라인 예측치

모든 주장은 프로젝트의 공개 검증 기록으로 거슬러 올라갑니다. 증명 표시 항목은 github.com/Wick-Lim/AIPU에서 재현할 수 있고(make q4k, make formal), 열림 표시 항목은 명시되고 기록된 남은 과제입니다. 처리량·BOM·좌석당 가격은 FPGA 데모가 나오기 전까지 추정치입니다. 남은 과제까지 정직하게 기록하는 원칙으로 만들었습니다. 여기 감춘 것은 없습니다.
